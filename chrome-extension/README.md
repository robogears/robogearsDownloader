# robogears Downloader — Chrome extension

Adds an `+ robogears` button to every track row on [open.spotify.com](https://open.spotify.com) — click and the track lands in the desktop app's queue (after a TIDAL match).

## Install (unpacked)

1. Make sure the **robogears Downloader** desktop app is installed and running.
2. In the app, open **Settings → Extension** and copy the **token** (32-char hex string).
3. In Chrome, go to `chrome://extensions/`.
4. Toggle **Developer mode** (top-right).
5. Click **Load unpacked**, then pick the `chrome-extension` folder inside this repo.
6. Click the extension's **Details → Extension options** (or right-click the extension icon → Options).
7. Paste the token, click **Save**, then **Test connection** to confirm.

That's it. Visit any Spotify page with a tracklist — playlists, albums, artist pages, search results, "Discover Weekly", whatever — and a small `+ robogears` button will appear at the end of each row.

## How it works

- Content script injects an `+ robogears` button into every `[data-testid="tracklist-row"]` it finds.
- Click → reads title + artist + Spotify track ID from the row's DOM.
- POSTs to `http://127.0.0.1:8273/queue/add` with the token in the `Authorization` header.
- The desktop app's local server takes the request, searches TIDAL for a match, and adds the matched track to the in-app queue.

The local server only listens on `127.0.0.1` (no LAN exposure). The token is generated by the app on first run and can be regenerated from the same Settings panel if it ever leaks.

## Limitations

- Only works while the desktop app is running.
- Spotify occasionally redesigns their tracklist DOM. If buttons stop appearing, the row selectors in `content.js` might need a refresh — `[data-testid="tracklist-row"]` has been stable for years but isn't guaranteed.
- The track has to be findable on TIDAL by title + artist search. ISRC-based lookups would be more reliable; can be added later if Spotify exposes ISRCs in the row's data.

## Files

- `manifest.json` — Manifest V3 declaration
- `content.js` — DOM injection + click handler
- `content.css` — button + toast styling
- `options.html` / `options.css` / `options.js` — token entry page
- `icon.png` — extension icon (reused from the desktop app)
